    Matreshka#on(names, callback, triggerOnInit=false, context) → matreshka

    Attaches event handler

    The method attaches event handler to Matreshka instance. Event could be fired by Matreshka#trigger method.

    Returns matreshka - self

    Arguments

    Name Default Type Details
    names eventNames

    Name of the space-delimited list of names (eg. "change:x ajaxcomplete change:y"). Look at eventNames docs to get more examples of event names.
    callback eventHandler

    A function to execute when the event is triggered
    triggerOnInit optional false boolean

    If triggerOnInit equals to true then an event handler will be triggered immediately
    context optional object

    An object to use as thiswhen executing callback

    Examples

    Basic usage

    this.on( 'customEvent', function() {
	alert( 'customEvent is fired' );
});
this.trigger( 'customEvent' );

    Passing context

    //Alert will be execuded in window context and display second argument,
//that has been passed to .trigger method
this.on( 'ohmygosh', alert, window );
this.trigger( 'ohmygosh', 'Hello world' );

    Calling event handler immediately after initialization

    //Alerts "bar" immediately and waits for triggering "foo" event
this.on( 'foo', function() {
	alert( 'bar' );
}, true );

    Attaches event handler to Matreshka instance passing debounced handler.

    This method limits the rate at which a handler can fire. Otherwise it works similar to Matreshka#on method

    Returns matreshka - self

    Arguments

    Name Default Type Details
    names eventNames

    Name of the space-delimited list of names (eg. "change:x ajaxcomplete change:y"). Look at eventNames docs to get more examples of event names.
    callback eventHandler

    A function to execute when the event is triggered
    debounceDelay optional 0 number

    Debounce delay
    triggerOnInit optional false boolean

    If triggerOnInit equals to true then an event handler will be triggered immediately
    context optional object

    An object to use as thiswhen executing callback

    Links

    • Matreshka#on
    • Matreshka.debounce
    • JavaScript Debounce Function

    Examples

    this.onDebounce( 'change:x', function() {
	alert( 'x = ' + this.x ); // x = 100
}, 300 );
this.x = 1;
for( var i = 0; i < 100; i++ ) {
	this.x++;
}

    Matreshka#off(names, callback, context) → matreshka

    Removes event handlers from Matreshka instance

    If you no longer need some event or few events, you can remove them by passing event names as first argument to the Matreshka#off method. You can specify the callback and context for the events you want to remove and you can pass nothing to remove all events.

    Returns matreshka - self

    Arguments

    Name Type Details
    names optional eventNames

    Space-delimited list of event names (e.g. "change:x ajaxcomplete change:y")
    callback optional eventHandler

    A function that has been passed to Matreshka#on
    context optional object

    An object that used as this when executing callback

    Examples

    Basic usage

    this.off( 'change:x bind' );

    Remove all events

    this.off();

    Remove event with given event handler

    var handler = function() { 
	//...
}
this.on( 'change:x', handler );
this.off( 'change:x', handler );

    Remove event with given event handler and given context

    var object = {};
this.on( 'change:x', handler, object );
this.off( 'change:x', handler, object );

    Matreshka#trigger(names, arg) → matreshka

    Fires event(s). Subsequent arguments will be passed to event handlers.

    After attaching event handler using Matreshka#on or Matreshka#once you can trigger it by Matreshka#trigger method and pass needed arguments to event handler by subsequent arguments. You can bind "all" event to catch any event triggering.

    Returns matreshka - self

    Arguments

    Name Type Details
    names optional eventNames

    Event name or space-delimited list of event names that you want to fire.
    arg optional *

    Arguments that you want to pass to event handlers

    Examples

    Basic usage

    this.on( 'jigurda ohyeah', function( a, b, c ) {
	alert( a, b, c );
});
this.trigger( 'ohyeah', 1, 2, 3 ); // alerts 6

    Matreshka#bindNode(key, node, binder, eventOptions) → matreshka

    Creates event bridge between DOM node and Matreshka instance property

    This powerful function binds to each other element and instance property with given options that say "when and how to extract element's value", "how to set element's value when property is changed".

    Returns matreshka - self

    Fires bind

    Arguments

    Name Type Details
    key string matreshka

    A key (or space-delimited list keys) that has to be bound to given element(s)
    node string node $nodes

    An element (DOM Node or DOM NodeList or array of nodes or balalaika array or jQuery instance or css selector...) that has to be bound to given key(s). Pay attention that element is required, otherwise the error will be thrown.
    binder optional binder

    A binder object which contains following properties: setValue (how to set value for an element), getValue (how to extract value from an element), on (when we have to extract a value from an element and assign it to given property) and initialize (what to do before binder initialized). Look at the binder documentation for more info.
    eventOptions optional eventOptions

    If you want to set "silent" flag or pass some options to a "bind" event handler

    Examples

    Basic usage 1

    this.bindNode( 'myKey', 'input[type="checkbox"]', {
	on: 'click',
	getValue: function() {
		return this.checked; // "this" is checkbox element  
	},
	setValue: function( v ) {
		this.checked = !!v;
	}
});
// now when element is bound you can change myKey and look at changes in UI
this.myKey = true; // makes checkbox checked
this.myKey = false; // makes checkbox unchecked

    Basic usage 2. Binding using Matreshka.defaultBinders array that contains binder for few types of DOM elements (input[type="text"], input[type="radio"], input[type="checkbox"], select, textarea). So you don't need to pass binder for such elements

    this.bindNode( 'myKey', '.checkbox' );

    Custom checkbox 1. This example shows how to create your own custom checkbox that has "checked" class if it's state is checked.

    this.bindNode( 'myKey', '.custom-checkbox', {
	on: 'click',
	getValue: function() {
		return $( this ).hasClass( 'checked' );
	},
	setValue: function( v ) {
		$( this ).toggleClass( 'checked', !!v );
	}
});

    Custom checkbox 2. In this example we'll do the same as in previous example but using predefined binder via Matreshka.defaultBinders.

    //shift means that we're adding new default binder to the beginning of MK.defaultBinders list
MK.defaultBinders.shift( function( element ) {
	if( $( element ).hasClass( 'custom-checkbox' ) ) return {
		on: 'click',
		getValue: function() {
			return $( this ).hasClass( 'checked' );
		},
		setValue: function( v ) {
			$( this ).toggleClass( 'checked', !!v );
		}
	};
});

this.bindNode( 'myKey', '.custom-checkbox' );

    "bind" event

    this.on( 'bind:myKey', function() { alert( 'ok!' ); });
this.bindNode( 'myKey', '.custom-checkbox' ); // alerts "ok!"

    "bind" event options

    this.on( 'bind:myKey', function() { alert( 'ok!' ); });
this.bindNode( 'myKey', '.custom-checkbox', {}, { silent: true } ); // no alert

    Extending default binders. For example we're working with input[type="text"]. By default "on" property for this element contains "keydown" string. But we want to use "blur" event for the element that has been bound to myKey property

    this.bindNode( 'myKey', '.my-input', { on: "blur" });

    Binding this to the element. If you want to use context (sandbox) for bindings of elements contained in single element, you can pass this special argument value to the method

    // you can use this.bindNode( '_this_', '.app' ); instead
this.bindNode( this, '.app' );
// this.$( '.my-element' ) takes element(s) from .app
this.bindNode( 'myKey', this.$( '.my-element' ) );

    "Missed bound elemen" error

    vae $el = $();
this.bindNode( 'x', $el ); // ERROR because $el is empty
// always check is element existing
if( $el.length ) {
	this.bindNode( 'x', $el );
}

    Matreshka#unbindNode(key, node, eventOptions) → matreshka

    Breaks bindings between property (or properties) and DOM element(s)

    If you no longer need a binding between element and Matreshka property you can remove it using this method.

    Returns matreshka - self

    Fires unbind

    Arguments

    Name Type Details
    key string null

    A key (or space-delimited list keys) that has to get rid of binding with given element(s) (null if you want to unbind element(s) from any key of instance)
    node optional string node $nodes

    An element (DOM Node or DOM NodeList or array of nodes or jQuery instance or css selector) that has to that has to get rid of binding with given key(s)
    eventOptions optional eventOptions

    If you want to set "silent" flag or pass some options to "unbind" event handlers

    Examples

    Basic usage

    this.bindNode( 'myKey', '.my-element' );
this.myKey = true; // changes myKey property and element state
this.unbindNode( 'myKey', '.my-element' );
this.myKey = false; // changes property only

    Matreshka#set(key, value, eventOptions)

    Sets value of given property and gives possibility to pass event object.

    Possible flags:

    • silent - don't fire change event
    • force - fire change event even if property value is not changed
    • forceHTML - change value for bound html element even if property value is not changed
    • skipMediator - skips mediator (look at Matreshka#mediate)

    Fires change change:KEY

    Arguments

    Name Type
    key string
    value *
    eventOptions optional eventOptions

    Examples

    Basic usage

    this.on( 'change:myKey', function( eventOptions ) {
	alert( eventOptions.value );
});
this.set( 'myKey', 3 ); // same as this[ 'myKey' ] = 3 or this.myKey = 3, alerts 3

    Passing eventOptions

    // no alert
this.set( 'myKey', 4, {
	silent: true
});
// alerts 5, eventOptions (first argument of event handler) contains property <code>"myFlag"</code>
this.set( 'myKey', 5, { 
	myFlag: 'Jigurda'
});

    binder: object

    binder contains information about how to extract value from an element, how to set value for an element and which element's event we have to listen

    Properties

    Name Type Details
    on optional string function

    event name or space-delimited list of events or which we have to listen. Function contained callback is also supported.

    getValue optional function

    function that tells how to extract value from an element (context this is given element)

    setValue optional function

    "How to set value" for an element (context this is given element)

    initialize optional function

    Fuction that executes once per binding

    Examples

    var binder = {
	on: 'click',
	getValue: function( options ) { return this.value; } 
	setValue: function( v, options ) { this.value = v; },
	initialize: function( options ) {
		alert( 'Binder is initialized. Initial input value=' + this.value );
	}
};
this.bindElement( 'a', '.my-checkbox', binder );
    var binder = {

		this.onclick = callback;
	},

	setValue: function( v, options ) { this.value = v; },
	initialize: function( options ) {
		alert( 'Binder is initialized. Initial input value=' + this.value );
	}
};
